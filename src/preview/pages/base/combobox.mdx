<Section title="Overview" id="overview">

The combobox primitives are intentionally generic — pass an `items` array
(any shape), a `getItemLabel` and `getItemKey`, and own the `searchValue` and
`selectedValue(s)` state. Optional `getItemGroup` collapses items into
sections; `applyButton` defers commit until the user clicks Apply;
`closeOnSelect` closes the popover after each pick. `creatable` + `onCreate`
turn it into a free-form input. Every callback (`onSearch`, `onLoadMore`,
`onCreate`) is an opt-in seam — no hardcoded fetcher, no opinionated wiring.

</Section>
